# 配置差异风险分析

## ⚠️ 发现的潜在风险

### 🔴 高风险差异

#### 1. 网络配置变化
**现有配置**: 使用默认Docker网络
**新配置**: 添加了自定义网络 `taofen-network`
**风险**: 可能导致容器间无法通信

#### 2. 健康检查URL变化  
**现有配置**: `http://localhost:8182/`
**新配置**: `http://localhost:8182/iiif/3/`
**风险**: 如果新URL不存在，容器会被标记为不健康

#### 3. Caddy卷映射增加
**现有配置**: 
```yaml
- ./Caddyfile:/etc/caddy/Caddyfile:ro
- caddy_data:/data
- caddy_config:/config  
- /srv/iiif/manifests:/srv/iiif/manifests:ro
```
**新配置**: 增加了
```yaml
- /var/www/taofen:/usr/share/caddy:ro
```
**风险**: 如果/var/www/taofen目录不存在，Caddy可能启动失败

### 🟡 中风险差异

#### 4. Cantaloupe环境变量
**现有**: 包含 `CANTALOUPE_CACHE_SERVER_DERIVATIVE: "FilesystemCache"`
**新配置**: 我遗漏了这个配置
**风险**: 缓存行为可能改变

### 🟢 安全的新增内容
- backend服务容器 (全新服务，不影响现有)
- 环境变量文件支持
- depends_on依赖关系

## 🛡️ 安全升级方案

### 方案1: 渐进式升级 (推荐)
1. **第一步**: 只添加backend服务，保持现有Caddy和Cantaloupe配置不变
2. **第二步**: 验证backend服务正常后，再逐步更新Caddy配置
3. **第三步**: 最后添加前端静态文件服务

### 方案2: 完全备份后升级
1. 备份当前工作的docker-compose.yml
2. 备份当前工作的Caddyfile
3. 快速替换并测试，如有问题立即回滚
