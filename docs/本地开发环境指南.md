# 韬奋文库本地开发环境指南

## 概述

本文档介绍如何在本地设置和运行韬奋文库的开发环境，包括Redis缓存服务的配置和使用。

## 系统要求

- Docker >= 20.10
- Node.js >= 18
- npm >= 8
- VS Code (推荐)

## 快速开始

### 1. 克隆项目

```bash
git clone <repository-url>
cd taofen_web
```

### 2. 启动开发环境

```bash
# 启动所有服务
./scripts/dev-start.sh
```

### 3. 访问服务

启动成功后，可以访问以下服务：

- **前端应用**: http://localhost:5173
- **缓存服务**: http://localhost:3002
- **AI后端**: http://localhost:3001
- **Redis Insight**: http://localhost:5540
- **IIIF服务**: http://localhost:8282

## 缓存服务

### 缓存策略

- **IIIF信息缓存**: 24小时
- **IIIF图像缓存**: 2小时
- **通用缓存**: 1小时（可配置）

### API接口

#### 基础缓存操作

```typescript
// 设置缓存
await cacheService.set('key', value, ttl);

// 获取缓存
const result = await cacheService.get('key');

// 删除缓存
await cacheService.delete('key');

// 批量获取
const results = await cacheService.mget(['key1', 'key2']);
```

#### IIIF缓存操作

```typescript
// 获取IIIF信息（自动缓存）
const iiifInfo = await IIIFUrlBuilder.getIIIFInfo(identifier);

// 获取IIIF图像（自动缓存）
const imageUrl = await IIIFUrlBuilder.getIIIFImage(identifier, {
  region: 'full',
  size: '400,600',
  rotation: '0',
  quality: 'default'
});

// 预取IIIF数据
await IIIFUrlBuilder.prefetchIIIFInfo(identifiers);

// 清空缓存
await IIIFUrlBuilder.clearIIIFCache();
```

### React Hooks

```typescript
// 使用缓存Hook
const { data, loading, error, refetch } = useCache('key', {
  ttl: 3600,
  refreshInterval: 60000
});

// 使用IIIF信息Hook
const { data: iiifInfo, loading: infoLoading } = useIIIFInfo(identifier);

// 使用缓存统计Hook
const { stats, loading: statsLoading } = useCacheStats();

// 使用批量缓存Hook
const { data: batchData } = useBatchCache(['key1', 'key2', 'key3']);
```

## 故障排除

### 常见问题

1. **端口冲突**
   ```bash
   # 查看端口占用
   lsof -i :5173
   # 停止占用进程
   kill -9 <PID>
   ```

2. **Redis连接失败**
   ```bash
   # 检查Redis状态
   docker-compose -f docker-compose.dev.yml logs redis
   # 重启Redis
   docker-compose -f docker-compose.dev.yml restart redis
   ```

3. **缓存服务启动失败**
   ```bash
   # 检查日志
   docker-compose -f docker-compose.dev.yml logs cache-service
   # 检查依赖
   docker-compose -f docker-compose.dev.yml ps
   ```

### 日志查看


```bash

```

## 最佳实践

### 缓存使用

1. **合理设置TTL**
   - 静态数据：长时间缓存
   - 动态数据：短时间缓存
   - 用户数据：根据业务需求

2. **缓存键命名**
   ```typescript
   // 使用冒号分隔的层级结构
   const userKey = `user:${userId}`;
   const iiifKey = `iiif:info:${identifier}`;
   ```

3. **错误处理**
   ```typescript
   try {
     const cached = await cacheService.get(key);
     if (cached) {
       return cached;
     }
     // 获取原始数据
     const data = await fetchOriginalData();
     // 设置缓存
     await cacheService.set(key, data, ttl);
     return data;
   } catch (error) {
     // 缓存失败时的处理
     console.error('缓存操作失败:', error);
     return fetchOriginalData();
   }
   ```

### 性能优化

1. **避免大对象**
   - 缓存必要的数据
   - 避免缓存过大的响应

2. **使用批量操作**
   - 减少网络请求
   - 提高效率

3. **监控和优化**
   - 定期检查缓存命中率
   - 优化缓存策略
