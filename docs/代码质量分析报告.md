### 3. 【功能】Redis 连接重连失效 🔄

**严重程度：** ⚠️⚠️⚠️ 最高

**文件位置：** `cache-service/lib/redis-cache.js:38-55`

```javascript
// ❌ 使用 once 而不是 on，导致重连后状态丢失
this.client.once('connect', () => {
  logger.info('Redis客户端连接成功');
  this.isConnected = true;
});

this.client.once('disconnect', () => {
  logger.warn('Redis客户端断开连接');
  this.isConnected = false;
});
```

**问题分析：**
- 使用 `once` 意味着只在第一次触发时执行
- 如果 Redis 重连（这在生产环境很常见），`isConnected` 状态将永远不会更新
- 导致所有后续的 `get/set/delete` 操作都会抛出 "Redis未连接" 错误
- 这是典型的"看起来正确但实际错误"的代码

**影响范围：**
- 🔌 Redis 重连后所有缓存操作失败
- 📉 服务降级到内存缓存
- 🚨 可能导致缓存不一致

**解决方案：**
```javascript
// ✅ 正确做法：使用 on 持续监听
this.client.on('connect', () => {
  logger.info('Redis客户端连接成功');
  this.isConnected = true;
});

this.client.on('disconnect', () => {
  logger.warn('Redis客户端断开连接');
  this.isConnected = false;
});
```

### 7. 【性能】Redis KEYS 命令导致阻塞 🚫

**严重程度：** ⚠️⚠️ 高

**文件位置：** `cache-service/lib/redis-cache.js:165-198`

```javascript
// ❌ KEYS 命令会阻塞整个 Redis 服务器
async clear(pattern = '*') {
  const keys = await this.client.keys(pattern);  // 危险！
  if (keys.length > 0) {
    const batchSize = 1000;
    for (let i = 0; i < keys.length; i += batchSize) {
      const batch = keys.slice(i, i + batchSize);
      const result = await this.client.del(batch);
    }
  }
}
```

**问题分析：**
- `KEYS *` 命令会**阻塞整个 Redis 服务器**，在数据量大时会导致超时
- Redis 文档明确警告不要在生产环境使用 KEYS
- 应该使用 `SCAN` 命令代替

**影响范围：**
- ⏱️ Redis 服务器阻塞
- 🐌 所有 Redis 操作变慢
- 💥 可能导致超时

**解决方案：**
```javascript
// ✅ 使用 SCAN 命令
async clear(pattern = '*') {
  let cursor = 0;
  const batchSize = 1000;

  do {
    const [nextCursor, keys] = await this.client.scan(cursor, {
      MATCH: pattern,
      COUNT: batchSize
    });

    if (keys.length > 0) {
      await this.client.del(keys);
    }

    cursor = nextCursor;
  } while (cursor !== 0);
}
```

**问题分析：**

- 没有指数退避重试
- 没有区分"服务关闭"和"临时网络错误"

**影响范围：**

- 📉 用户体验下降

**解决方案：**
```typescript
// ✅ 添加重试机制
private async checkHealth(retryCount = 0): Promise<void> {
  try {
    const response = await fetch(`${this.baseUrl.replace('/cache', '')}/health`);
    this.isConnected = response.ok;
    this.retryCount = 0; // 重置重试计数
  } catch {
    if (retryCount < 3) {
      // 指数退避重试
      const delay = Math.pow(2, retryCount) * 1000;
      setTimeout(() => this.checkHealth(retryCount + 1), delay);
    } else {
      this.isConnected = false;
    }
  }
}
```

### 13. 【内存】内存缓存泄漏风险 💾

**严重程度：** ⚠️⚠️ 中

**文件位置：** `backend/server.js:90-105`

```javascript
// ❌ setTimeout 回调不会被阻止
async function setToCache(key, value, ttl = 3600) {
  try {
    const serializedValue = JSON.stringify(value);
    if (redisClient && redisClient.isReady) {
      await redisClient.setEx(key, ttl, serializedValue);
    } else {
      memoryCache.set(key, value);
      // ⚠️ 内存缓存设置过期时间
      setTimeout(() => memoryCache.delete(key), ttl * 1000);
    }
  }
}
```

**问题分析：**
- setTimeout 回调不会被阻止，即使 Redis 后来连接成功
- 如果频繁切换缓存策略，会创建大量未清理的定时器
- 内存泄漏：定时器持有 key 引用，即使数据已删除

**影响范围：**
- 💾 长期运行会导致内存泄漏
- 📉 性能下降
- 🐛 可能导致 OOM

**解决方案：**
```javascript
// ✅ 存储定时器引用，允许取消
const cacheTimers = new Map();

async function setToCache(key, value, ttl = 3600) {
  try {
    const serializedValue = JSON.stringify(value);
    if (redisClient && redisClient.isReady) {
      await redisClient.setEx(key, ttl, serializedValue);
      // 清除旧的定时器
      if (cacheTimers.has(key)) {
        clearTimeout(cacheTimers.get(key));
        cacheTimers.delete(key);
      }
    } else {
      memoryCache.set(key, value);
      const timer = setTimeout(() => {
        memoryCache.delete(key);
        cacheTimers.delete(key);
      }, ttl * 1000);
      cacheTimers.set(key, timer);
    }
  }
}
```

- 每个键都单独查询，网络往返次数过多
- cache-service 正确使用了 mget

**影响范围：**
- 🐌 性能差：N 个键需要 N 次网络往返
- ⏱️ 延迟高
- 📈 服务器负载增加

**解决方案：**
```javascript
// ✅ 使用 MGET 批量查询
app.post('/api/cache/mget', async (req, res) => {
  const { keys } = req.body;

  if (!Array.isArray(keys) || keys.length === 0) {
    return res.status(400).json({ error: '键数组不能为空' });
  }

  try {
    // 使用 MGET 一次性查询所有键
    const values = await redisClient.mGet(keys);
    const results = keys.map((key, index) => ({
      key,
      value: values[index] ? JSON.parse(values[index]) : null,
      found: values[index] !== null
    }));

    res.json({ success: true, results });
  } catch (error) {
    res.status(500).json({ success: false, error: error.message });
  }
});
```

  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);
  const [data, setData] = useState<T | null>(null);
