## 数据流程图（实际资源限制下的优化流程）

```mermaid
sequenceDiagram
    participant U as 用户
    participant F as 前端应用
    participant A as API网关
    participant R as 资源管理器
    participant C as 缓存服务
    participant S as 业务服务
    participant D as 数据库
    participant E as 外部API
    participant L as 限流控制
    
    U->>F: 访问页面
    F->>F: 资源检查(2核2G)
    F->>A: 请求资源
    
    A->>L: 限流检查
    alt 超出限制
        L-->>A: 返回429错误
        A-->>F: 错误提示
        F-->>U: 显示加载中
    else 正常访问
        A->>C: 查询缓存
        alt 缓存命中
            C-->>A: 返回缓存数据
            A-->>F: 返回数据
            F-->>U: 渲染页面
        else 缓存未命中
            A->>R: 资源检查
            alt 资源不足
                R-->>A: 返回503错误
                A-->>F: 服务繁忙提示
                F-->>U: 显示稍后重试
            else 资源充足
                A->>S: 业务逻辑处理
                S->>D: 查询数据库
                D-->>S: 返回数据
                S->>E: 调用外部API
                Note over S,E: AI服务响应15-30秒
                E-->>S: 返回结果
                S->>C: 更新缓存
                S-->>A: 返回处理结果
                A-->>F: 返回数据
                F-->>U: 渲染页面
            end
        end
    end
    
    Note over U,L: 1M带宽限制影响响应速度
    Note over R,S: 2核2G限制并发处理能力
```
