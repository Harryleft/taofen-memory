# AI解读功能实现计划

## 需求概述

在 `HandwritingLightbox.tsx` 组件中添加AI解读功能，让用户可以查看AI对古文内容的通俗解读。

### 核心功能
- 在原文旁边添加AI解读按钮
- 点击后调用GLM-4.5 API生成通俗易懂的解读
- 支持原文与AI解读之间的切换显示
- 提供友好的加载状态和错误处理

### 技术规格
- AI模型：GLM-4.5
- API密钥：2cdde2240d0a446b9bd7962a8c5a25fe.suOORlOs7kv84ZEF（一个月后失效，无限额度）
- 角色设定：优秀讲解员
- 提示词设计：基于时间、标题、原文、注释信息

## Linus式需求分析

### 核心判断
✅ **值得做**：为手写内容添加AI解读功能有明确的用户价值，技术实现可行，风险可控。

### 关键洞察
- **数据结构**：需要在TransformedHandwritingItem基础上增加AI解读状态
- **复杂度**：主要是网络请求管理和UI状态切换，没有本质复杂性
- **风险点**：API调用失败处理和并发控制

### Linus式方案
1. **简化数据结构**：将AI解读视为原文的另一种表示形式
2. **消除特殊情况**：通过状态管理避免并发问题
3. **最清晰实现**：直接在原文区域添加切换功能
4. **零破坏性**：保留所有现有功能，只添加新功能

## 技术实现细节

### 状态管理设计
```typescript
interface AIInterpretationState {
  content: string;        // AI解读内容
  isLoading: boolean;     // 加载状态
  error: string | null;   // 错误信息
  isViewingAI: boolean;   // 当前是否查看AI解读
}
```

### API调用设计
```typescript
interface AIInterpretationRequest {
  title: string;
  content: string;
  notes: string;
  time: string;
}

interface AIInterpretationResponse {
  interpretation: string;
  success: boolean;
  error?: string;
}
```

### 提示词模板
```
角色：你是一名优秀的邹韬奋事迹讲解员，擅长将当时邹韬奋写的手稿信息转换为让现代人容易理解的语言。

任务：请对以下手稿内容进行通俗易懂的解读。

上下文信息：
- 时间：{time}
- 标题：{title}
- 原文：{content}
- 注释：{notes}

输出要求：
1. 用自然语言段落重新表述原文含义
2. 解释重要的时代、文化背景
3. 保持语言生动有趣，避免学术化表达
4. 解读长度控制在原文的1.5倍以内
```

## 风险评估与应对

### 技术风险
1. **API密钥失效**
   - 风险：一个月后API密钥失效
   - 应对：设计配置化的API密钥管理

2. **网络请求失败**
   - 风险：用户网络环境不稳定
   - 应对：实现重试机制和离线提示

3. **并发调用**
   - 风险：用户快速多次点击
   - 应对：请求防抖和按钮禁用

### 用户体验风险
1. **AI解读质量**
   - 风险：AI解读不够准确或有用
   - 应对：提供"重新生成"功能和反馈渠道

2. **加载时间过长**
   - 风险：用户等待失去耐心
   - 应对：优化加载动画和进度提示
